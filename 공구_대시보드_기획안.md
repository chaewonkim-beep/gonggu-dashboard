# 달콤수학 공구 관리 대시보드 기획안

> 작성일: 2026-06-12 · 개정일: 2026-06-18
> 담당: 공구 관리팀

---

## 1. 역할 (Role)

> **핵심 질문: 오늘 어떤 공구 업무를 먼저 처리해야 하는가?**

공구 담당자가 매일 업무 시작 시 이 질문에 즉시 답할 수 있도록,
**오늘 처리할 것 · 이번 주 처리할 것 · 공구사별 이슈**를 한 화면에서 구분해 보여 주는 **업무용 단일 페이지 대시보드**.

| 구분 | 내용 |
|------|------|
| 사용자 | 달콤수학 공구 담당자 |
| 사용 맥락 | 매일 업무 시작 시 우선순위 파악 / 공구사 미팅·소통 직후 기록 |
| 우선순위 구분 | 오늘 처리 → 이번 주 처리 → 공구사 이슈 (3단계) |
| 운영 환경 | 로컬 HTML 파일 (브라우저 단독 실행, 서버 불필요) |

---

## 2. 목표 (Goal)

### 핵심 목표
- 화면에 진입하는 순간 **오늘 처리할 업무**가 즉시 보인다
- 할 일·일정·소통 기록이 **서로 연결**되어 누락을 방지한다
- 소통 기록이 **후속 할 일로 자동 이어진다**
- 업무 기록은 삭제하지 않고 **완료·보관 처리**로 영속 보존한다

### 세부 목표

| # | 목표 | 측정 기준 |
|---|------|-----------|
| G1 | 진행중 공구사 진행률 시각화 | 바오밥·칠진사 각 진행률(%) + 단계 체크리스트 표시 |
| G2 | 오늘·이번 주·이슈 우선순위 구분 | 3단계 뷰 (오늘 처리 / 이번 주 처리 / 공구사 이슈) |
| G3 | 소통 이력 누적 관리 | 공구사·유형별 필터 + 날짜순 피드 |
| G4 | 할 일 추가·완료·보관 | 긴급/일반 구분, localStorage 영속 저장, 삭제 없음 |
| G5 | 단일 파일 운영 | 외부 서버·DB 없이 브라우저만으로 완전 동작 |
| G6 | 할 일 ↔ 일정 연결 | 할 일 추가 시 공구사 + 연결 일정 선택, 일정 카드에 연결 건수 표시 |
| G7 | 소통 기록 → 후속 할 일 자동 연결 | 소통 저장 시 후속 할 일 체크 → 할 일 자동 생성 + 양방향 링크 |

## 3. 입력 (Input)

### 3-1. 정적 초기 데이터 (HTML 내 하드코딩)

**공구사 (5개) — 이벤트와 분리하여 정의**

| 공구사 ID | 공구사명 |
|-----------|---------|
| navita | 나비타 |
| chilgeun | 칠진사 |
| naelna | 낼나 |
| baobab | 바오밥 |
| jnj | 제이앤제이 |

**개별 공구/이벤트 (공구사 ID 참조, 이벤트 ID 부여)**

| 이벤트 ID | 공구사 | 일정 | 내용 |
|-----------|--------|------|------|
| evt-001 | baobab | 2026-06-15 | 바오밥 공구 오픈 |
| evt-002 | chilgeun | 2026-07-29 | 칠진사 공구 |
| evt-003 | naelna | 2026-08-03 | 소마셈·팩토 공구 시작 |
| … | … | … | … |

> 공구사는 여러 번의 이벤트를 가질 수 있으므로 분리 관리합니다.
> 이벤트 ID는 할 일·소통 기록에서 참조 키로 사용합니다.

**기타 초기 데이터**

| 데이터 | 항목 예시 |
|--------|-----------|
| 기본 할 일 | 칠진사 미팅 준비, 낼나 메일 답변, 바오밥 오픈 확인 등 |
| 기본 소통내역 | 공구사별 최근 소통 이력 (날짜·유형·내용) |
| 진행 단계 | 바오밥 6단계, 칠진사 5단계 (체크리스트 형태) |

### 3-2. 사용자 실시간 입력

| 입력 유형 | 기존 필드 | 추가/변경 필드 | 저장 위치 |
|-----------|-----------|----------------|-----------|
| 할 일 추가 | 공구사, 긴급/일반, 내용 | **연결 일정 (이벤트 ID 선택, 선택 사항)** | `localStorage` → `tasks` |
| 소통내역 추가 | 공구사, 유형, 내용 | **후속 할 일 여부 (체크박스 + 내용 입력란)** | `localStorage` → `communications` |
| 진행 단계 체크 | 바오밥·칠진사 각 단계 완료 표시 | — | `localStorage` → `progressSteps` |
| 할 일 완료 체크 | 개별 항목 완료 처리 | — | `localStorage` → `tasks` |
| 할 일 **보관** | ~~삭제~~ | **보관 처리 (archived: true)** | `tasks`에서 상태 변경 |

### 3-3. localStorage 데이터 구조

```
vendors[]          — 공구사 마스터 (id, name)
events[]           — 개별 공구/이벤트 (id, vendorId, date, title, status)
tasks[]            — 할 일 (id, vendorId, eventId?, content, urgent, done, archived)
communications[]   — 소통 기록 (id, vendorId, date, type, content, followUpTaskId?)
progressSteps      — 진행 단계 체크 상태 (vendorId → stepIndex → checked)
```

### 3-4. 시스템 입력
- 오늘 날짜 (JavaScript `new Date()` 기반 D-Day 자동 계산)
- 브라우저 `localStorage` (이전 세션 데이터 자동 복원)

## 4. 출력 (Output)

### 4-1. 화면 구조: 왼쪽 목차 + 우측 콘텐츠

```
┌────────────────┬──────────────────────────────────────────┐
│  왼쪽 목차      │  헤더: 달콤수학 공구 관리 / 오늘 날짜 표시    │
│                ├──────────────────────────────────────────┤
│  ① 오늘 확인   │  ① 오늘 확인  ← 첫 화면 진입 시 기본 섹션   │
│  ② 공구 일정   │    [오늘 처리] [이번 주 처리] [공구사 이슈]  │
│  ③ 공구사 현황 ├──────────────────────────────────────────┤
│  ④ 할 일      │  ② 공구 일정                               │
│  ⑤ 소통 기록  │    캘린더 + 일정 카드 (D-Day, 연결 할 일 수) │
│                ├──────────────────────────────────────────┤
│                │  ③ 공구사별 현황                           │
│                │    공구사 카드 클릭 → 상세 패널 펼침         │
│                │    패널: 진행 단계 / 다음 일정 / 최근 소통 / │
│                │          남은 할 일                        │
│                ├──────────────────────────────────────────┤
│                │  ④ 할 일                                  │
│                │    공구사·연결 일정 필터 / 목록 / 추가 폼    │
│                ├──────────────────────────────────────────┤
│                │  ⑤ 소통 기록                              │
│                │    유형별 필터 / 피드 / 후속 할 일 포함 추가 폼│
└────────────────┴──────────────────────────────────────────┘
```

### 4-2. ① 오늘 확인 섹션 (핵심 화면)

3개 구역으로 분리하여 우선순위를 즉시 인지:

| 구역 | 표시 내용 |
|------|-----------|
| **오늘 처리** | D-0 일정에 연결된 할 일 + 미완료 긴급 할 일 전체 |
| **이번 주 처리** | D-1~D-7 일정 + 일반 미완료 할 일 (긴급 제외) |
| **공구사 이슈** | 소통 기록 유형이 '이슈'인 항목의 후속 할 일 중 미완료 건 |

### 4-3. ② 공구 일정 — 일정 카드 연결 항목 표시

각 일정 카드 하단에 **연결된 할 일 N건 / 관련 소통 N건** 접기/펼치기 제공.
펼치면 해당 할 일·소통 기록 인라인 표시.

### 4-4. ③ 공구사별 현황 — 상세 패널

공구사 카드 클릭 시 우측 또는 하단에 상세 패널 펼침:

```
┌─────────────────────────────────┐
│  바오밥  진행률 80%  ██████████░░ │
│  진행 단계:  ✅계약 ✅발주 ✅촬영 … │
├─────────────────────────────────┤
│  다음 일정: 6/15 오픈 (D-3)       │
├─────────────────────────────────┤
│  최근 소통: 6/12 담당자 확인 완료   │
├─────────────────────────────────┤
│  남은 할 일: 오픈 확인, 링크 공유   │
└─────────────────────────────────┘
```

### 4-5. ④ 할 일 추가 폼 — 연결 일정 선택 추가

```
공구사: [선택 ▼]   연결 일정: [선택 ▼]   긴급 [●] 일반 [○]
내용: [___________________________]   [추가]
```

- 연결 일정은 선택 사항 (선택 안 하면 공구사에만 연결)
- 일정 선택 시 해당 이벤트 카드의 연결 건수 자동 증가

### 4-6. ⑤ 소통 기록 추가 폼 — 후속 할 일 연결

```
공구사: [선택 ▼]   유형: [선택 ▼]
내용: [_________________________________]

□ 후속 할 일 만들기
  └ 할 일 내용: [___________________]  긴급 [●] 일반 [○]

[저장]
```

- 체크박스 ON 시 할 일 입력란 노출
- 저장 시 소통 기록 + 할 일 동시 생성, 양방향 링크(followUpTaskId) 기록

### 4-7. 할 일 삭제 → 보관 처리

- `×` 삭제 버튼 제거 → **`보관` 버튼**으로 교체
- 보관된 항목은 목록에서 숨김
- 할 일 목록 하단 **'보관함 보기' 토글**로 확인 가능
- 기본 제공 할 일도 보관 처리 가능 (완전 삭제 불가)

### 4-8. 상태별 시각 출력

| 상태 | 표시 방식 |
|------|-----------|
| 오늘 일정 | 오렌지 배지 + 헤더 긴급 표시 |
| 진행중 | 옐로우 배지 |
| 예정 | 브라운 계열 배지 + D-N 표시 |
| 이슈 | 레드 배지 + 소통내역 강조색 |
| 완료 | 회색 배지 + 취소선 |
| 보관 | 연한 회색, 목록에서 숨김 |
| 긴급 할 일 | 오렌지-레드 배경 + 긴급 라벨 |

### 4-9. 파일 산출물

| 파일 | 설명 |
|------|------|
| `공구_대시보드.html` | 단일 HTML 파일 (CSS·JS 내장, 브라우저 직접 실행) |

## 5. 제약 (Constraints)

### 기술 제약

| 항목 | 내용 |
|------|------|
| 실행 환경 | 서버 없이 브라우저 단독 실행 (file:// 프로토콜) |
| 외부 의존성 | Google Fonts (Noto Sans KR) CDN 1개만 허용 |
| 데이터 저장 | `localStorage`만 사용 (용량 ~5MB, 브라우저·기기 종속) |
| 데이터 공유 | 기기 간 동기화 없음 (개인 PC 전용) |
| 백업 | 수동 파일 복사만 가능 (자동 백업 없음) |

### 운영 제약

| 항목 | 내용 |
|------|------|
| 공구사 수 | 초기 5개사 고정 (추가 시 HTML 직접 수정 필요) |
| 진행 단계 | 초기 설정값 HTML 하드코딩 (수정 시 파일 편집) |
| 할 일 삭제 | **삭제 없음 — 완료 또는 보관 처리만 가능, 운영 기록 영속 보존** |
| 날짜 기준 | `new Date()` 자동 적용 |
| 구글 캘린더 | 현재 미연동 (향후 ICS 내보내기 또는 API 연동 검토) |

## 6. 검수 항목 (QA Checklist)

### 6-1. 오늘 확인 섹션

- [ ] 화면 진입 시 기본으로 ① 오늘 확인 섹션 표시
- [ ] "오늘 처리" 구역: D-0 일정 연결 할 일 + 긴급 미완료 할 일 정상 표시
- [ ] "이번 주 처리" 구역: D-1~D-7 일정 + 일반 미완료 할 일 정상 표시
- [ ] "공구사 이슈" 구역: 이슈 소통 기록의 후속 할 일 미완료 건 정상 표시

### 6-2. 진행 현황

- [ ] 바오밥·칠진사 단계 클릭 시 체크 상태 토글
- [ ] 단계 완료 시 진행률(%) 및 막대 그래프 즉시 업데이트
- [ ] 새로고침 후 진행 단계 상태 유지

### 6-3. 할 일 관리

- [ ] 할 일 추가 폼에서 공구사 + 연결 일정 선택 가능
- [ ] 연결 일정 선택 후 저장 → 해당 일정 카드 "연결된 할 일 N건" 카운트 증가
- [ ] Enter 키로 할 일 추가 동작
- [ ] 긴급/일반 토글 선택 반영
- [ ] `보관` 버튼 클릭 → 목록에서 숨김
- [ ] '보관함 보기' 토글 → 보관 항목 확인
- [ ] `×` 삭제 버튼 없음 (완전 삭제 불가 확인)
- [ ] 체크박스 클릭 시 완료 처리 (취소선 + 하단 이동)
- [ ] 새로고침 후 추가·완료·보관 상태 유지

### 6-4. 소통 기록

- [ ] 공구사·유형 선택 후 내용 입력 → 저장 시 피드 최상단에 표시
- [ ] "후속 할 일 만들기" 체크 → 할 일 입력란 노출
- [ ] 후속 할 일 포함 저장 → 소통 기록 + 할 일 동시 생성
- [ ] 생성된 할 일에서 소통 기록 링크 확인 (양방향)
- [ ] 유형 필터 (전체/이슈/할 일/완료/확인/정보) 정상 작동
- [ ] 새로고침 후 추가 소통내역 유지

### 6-5. 공구 일정

- [ ] 일정 카드 하단 "연결된 할 일 N건 / 관련 소통 N건" 표시
- [ ] 접기/펼치기 클릭 시 연결 항목 인라인 표시
- [ ] `‹` `›` 버튼으로 월 이동
- [ ] 오늘 날짜 하이라이트 표시

### 6-6. 공구사 상세 패널

- [ ] 공구사 카드 클릭 → 상세 패널 펼침 (진행 단계 / 다음 일정 / 최근 소통 / 남은 할 일)
- [ ] 동일 카드 재클릭 → 패널 닫힘

### 6-7. 데이터 영속성

- [ ] 브라우저 새로고침 후 전체 상태 복원
- [ ] 브라우저 탭 닫고 재오픈 후 상태 복원

### 6-8. 오류 케이스

- [ ] 할 일 내용 없이 추가 시 토스트 경고, 추가 안 됨
- [ ] 소통내역 공구사 미선택 후 추가 시 토스트 경고
- [ ] 후속 할 일 체크 후 내용 비워 저장 시 경고

## 9. Claude Code 요청문

```
공구 담당자가 매일 업무 시작 시 "오늘 어떤 공구 업무를 먼저 처리해야 하는가?"를
즉시 파악할 수 있는 단일 HTML 대시보드를 만들어 주세요.

【레이아웃】
왼쪽에 고정 목차(오늘 확인 / 공구 일정 / 공구사별 현황 / 할 일 / 소통 기록),
오른쪽에 섹션 콘텐츠를 표시해 주세요.

【① 오늘 확인 섹션】
세 구역으로 나눠 주세요:
- 오늘 처리: D-0 일정 연결 할 일 + 미완료 긴급 할 일
- 이번 주 처리: D-1~D-7 일정 + 일반 미완료 할 일
- 공구사 이슈: 이슈 유형 소통 기록의 미완료 후속 할 일

【② 공구 일정 섹션】
월간 캘린더와 일정 카드 목록을 보여 주세요.
각 일정 카드에는 D-Day와 "연결된 할 일 N건 / 관련 소통 N건"을
접기/펼치기로 표시해 주세요.

【③ 공구사별 현황 섹션】
공구사 카드 클릭 시 상세 패널을 펼쳐 주세요.
패널에는 진행 단계 체크리스트, 다음 일정(D-Day), 최근 소통 내용,
남은 할 일이 한 화면에 통합되어 보여야 합니다.

【④ 할 일 섹션】
할 일 추가 폼에 공구사 선택과 연결 일정 선택(선택 사항)을 포함해 주세요.
할 일은 삭제하지 않고 완료 또는 보관 처리만 가능하게 해 주세요.
'보관함 보기' 토글로 보관된 항목을 별도 확인할 수 있게 해 주세요.

【⑤ 소통 기록 섹션】
소통 기록 추가 폼에 "후속 할 일 만들기" 체크박스를 포함해 주세요.
체크 시 할 일 내용 입력란이 나타나고, 저장하면 소통 기록과 할 일이
동시에 생성되어 양방향으로 연결되어야 합니다.

【데이터】
localStorage를 사용하되, 공구사(vendors) / 이벤트(events) / 할 일(tasks) /
소통 기록(communications) / 진행 단계(progressSteps)를 구분해서 저장해 주세요.
이벤트에는 고유 ID를 부여하고, 할 일과 소통 기록에서 eventId로 참조할 수 있게 해 주세요.

【디자인】
달콤교육 브랜드에 맞게 크림/화이트(#FFFDF7) 배경에
옐로우(#F5C842) · 브라운(#7B5C3E) · 오렌지(#F08030) 포인트 컬러를 사용해 주세요.
카드와 상세 패널 중심 레이아웃으로 구성하고, 이모지는 최소화해 주세요.
MacBook과 iPad(768px 이상)에서 잘 보이도록 해 주세요.
```
